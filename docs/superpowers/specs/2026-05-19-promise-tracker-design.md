# 선출직 공약 확인 앱 MVP 설계

## 결론

첫 버전은 CSV 파일로 데이터를 관리하는 선출직 공약 확인 웹앱으로 만든다.

첫 구현은 국회의원부터 시작한다. 다만 데이터 구조는 2026년 6월 3일 제9회 전국동시지방선거 당선자도 나중에 추가할 수 있게 만든다.

사용자는 지역명 또는 이름으로 선출직 공직자를 찾고, 해당 인물의 공약별 이행 상태와 근거 링크를 확인한다. 운영자는 별도 관리자 페이지 없이 CSV 파일을 수정해 데이터를 관리한다.

## 목표

- 국민이 내 지역 선출직 공직자의 공약을 쉽게 찾는다.
- 공약별로 현재 상태를 확인한다.
- 상태 판단의 근거 링크를 함께 공개해 신뢰를 높인다.
- 1인 운영자가 엑셀 또는 CSV로 관리할 수 있게 한다.
- 국회의원에서 시작하되 지방선거 당선자로 확장할 수 있게 한다.

## 첫 버전 범위

### 포함할 기능

1. 선출직 공직자 검색
   - 이름으로 검색한다.
   - 지역 또는 지역구로 검색한다.
   - 첫 구현 데이터는 국회의원으로 제한한다.

2. 상세 화면
   - 이름을 보여준다.
   - 선거 종류를 보여준다.
   - 직책을 보여준다.
   - 정당을 보여준다.
   - 지역 또는 지역구를 보여준다.
   - 공약 목록을 보여준다.
   - 공약별 상태를 보여준다.
   - 공약별 근거 링크를 보여준다.
   - 마지막 확인일을 보여준다.

3. CSV 데이터 관리
   - 운영자는 CSV 파일을 수정한다.
   - 앱은 CSV 파일을 읽어 화면에 보여준다.
   - 첫 버전에는 로그인 관리자 페이지를 만들지 않는다.

### 제외할 기능

- 관리자 로그인
- 앱 안에서 공약 직접 입력
- AI 자동 분석
- 첫 구현에서 지방선거 당선자 전체 관리
- 모든 국회의원 또는 지방선거 당선자 데이터 자동 수집
- 사용자 회원가입
- 댓글, 신고, 토론 기능

## 사용자 흐름

1. 사용자가 앱 첫 화면에 들어온다.
2. 검색창에 이름 또는 지역명을 입력한다.
3. 검색 결과에서 인물을 선택한다.
4. 상세 화면에서 공약 목록을 본다.
5. 공약 상태와 근거 링크를 확인한다.
6. 근거 링크를 눌러 원문 자료를 확인한다.

## 화면 구성

### 첫 화면

- 서비스 이름
- 검색창
- 검색 예시
- 선거 종류 필터
- 최근 확인된 인물 또는 샘플 인물 목록

### 검색 결과 화면

- 이름
- 선거 종류
- 직책
- 정당
- 지역 또는 지역구
- 상세 보기 버튼

### 상세 화면

- 기본 정보
- 공약 상태 요약
- 공약 목록
- 각 공약의 상태, 근거 링크, 마지막 확인일, 메모

## 공약 상태

첫 버전의 상태값은 다음 4개로 제한한다.

- 완료: 공약이 실제로 이행되었다고 볼 근거가 있는 상태
- 진행 중: 법안 발의, 예산 반영, 사업 추진 등 진행 근거가 있는 상태
- 미이행: 확인 가능한 이행 근거가 없거나 반대되는 근거가 있는 상태
- 확인 필요: 아직 판단하기 어려운 상태

## CSV 형식

CSV 파일은 초보자도 엑셀에서 수정할 수 있는 구조로 둔다.

```csv
선거종류,직책,이름,정당,지역,공약명,상태,근거링크,마지막확인일,메모
국회의원선거,국회의원,홍길동,예시당,서울 강남구갑,청년 주거 지원 확대,진행 중,https://example.com,2026-05-19,예산안 발의 확인
지방선거,시장,김예시,예시당,서울특별시,대중교통 요금 부담 완화,확인 필요,https://example.com,2026-06-10,당선 후 추진 여부 확인 필요
```

### 필수 항목

- 선거종류
- 직책
- 이름
- 정당
- 지역
- 공약명
- 상태
- 근거링크
- 마지막확인일

### 선택 항목

- 메모

## 데이터 원칙

- 비밀번호, API Key, 고객 개인정보는 저장하지 않는다.
- 공약 상태에는 반드시 근거 링크를 붙인다.
- 근거가 약하면 완료 또는 미이행으로 단정하지 않고 확인 필요로 둔다.
- 사용자가 원문을 직접 확인할 수 있게 링크를 공개한다.
- 선거 전에는 후보자 또는 공약 자료를 모을 수 있지만, 이 앱의 공약 이행 확인 대상은 당선자 기준으로 둔다.

## 공식 자료 후보

- 중앙선거관리위원회 정책·공약마당: 선거 공약 원문 확인 후보
- 공공데이터포털 국회의원 정보 통합 API: 의원 기본 정보 확인 후보
- 열린국회정보 또는 국회 공개 자료: 이후 의정활동 확인 후보
- 중앙선거관리위원회 제9회 전국동시지방선거 일정: 지방선거 일정 확인 후보

첫 버전은 API 자동 연동을 하지 않고, CSV에 필요한 내용을 직접 넣는다. 자동 연동은 MVP 이후 확장 기능으로 둔다.

## 기술 방향

첫 버전은 정적 웹앱 또는 간단한 프론트엔드 앱으로 만든다.

- CSV 파일을 앱 안에 둔다.
- 앱이 CSV를 읽어 검색과 상세 화면을 구성한다.
- 서버와 데이터베이스는 처음에는 쓰지 않는다.

이 방식은 빠르게 만들 수 있고, 1인 운영자가 파일 하나로 데이터를 관리할 수 있다.

## 나중에 확장할 기능

1. 관리자 페이지
   - 앱 안에서 의원과 공약을 직접 입력한다.

2. 데이터베이스
   - 공약 수가 많아지면 CSV 대신 데이터베이스를 사용한다.

3. AI 검토 보조
   - 근거 자료를 읽고 공약 상태 초안을 만든다.
   - 최종 판단은 운영자가 확인한다.

4. 지방선거 당선자 확장
   - 2026년 6월 3일 제9회 전국동시지방선거 이후 당선자를 추가한다.
   - 먼저 시·도지사, 시장·군수·구청장부터 넣는다.
   - 광역의원, 기초의원, 교육감은 이후 단계로 넓힌다.

## 검증 기준

MVP가 완료되었다고 볼 기준은 다음과 같다.

- 사용자가 이름으로 의원을 찾을 수 있다.
- 사용자가 지역 또는 지역구로 의원을 찾을 수 있다.
- 상세 화면에서 공약 목록을 볼 수 있다.
- 공약마다 상태와 근거 링크가 보인다.
- CSV 파일을 수정하면 화면 데이터가 바뀐다.
- 개인정보나 비밀값을 저장하지 않는다.
- CSV 구조에 지방선거 당선자 데이터를 추가해도 같은 화면 구조로 보여줄 수 있다.

## 남은 결정 사항

구현 전에 다음을 정하면 된다.

- 앱 이름
- 첫 샘플 데이터에 넣을 의원 수
- 지방선거 확장 시 첫 대상 범위
- 디자인 분위기
- 배포 방식
